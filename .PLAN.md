# Chain Monitor Comprehensive Refactoring Plan
## Quality: 9/10 | Integrity: 9/10 | Maintainability: 9/10

### Architectural Principles
1. **No file-I/O as API boundaries.** Data flows through function calls, not JSON files.
2. **Fail-safe defaults.** Pipeline continues with best-effort data; never crash on optional stages.
3. **Explicit side effects.** All file writes, network calls, subprocesses are isolated and logged.
4. **Schema-driven.** Core types validated with Pydantic at pipeline boundaries.
5. **Lifecycle management.** All resources (sessions, executors, browsers) have deterministic cleanup.

### C2. Add atomic file write utility
```python
# processors/pipeline_utils.py
def atomic_write_json(path: Path, data: dict) -> None:
    """Write JSON atomically via temp file + rename."""
    tmp = path.with_suffix(path.suffix + ".tmp")
    with open(tmp, "w", encoding="utf-8") as f:
        json.dump(data, f, indent=2)
    tmp.replace(path)

async def atomic_write_text(path: Path, content: str) -> None:
    """Write text atomically via temp file + rename."""
    import aiofiles
    tmp = path.with_suffix(path.suffix + ".tmp")
    async with aiofiles.open(tmp, "w", encoding="utf-8") as f:
        await f.write(content)
    tmp.replace(path)
```

- `_save_run_log()` in main.py â†’ `atomic_write_json()`

### C3. Fix ThreadPoolExecutor leak in parallel_runner.py
**Current:** Module-level executor, never shut down:
```python
_EXECUTOR = ThreadPoolExecutor(max_workers=8, thread_name_prefix="collector-")
```

Fix options:
Option A (simple): Make `_EXECUTOR` a function-level default arg, or pass it into `collect_all()` and let caller manage lifecycle.
Option B (better): Don't use ThreadPoolExecutor at all â€” `TwitterCollector` already uses `ProcessPoolExecutor`, and all other collectors are sync HTTP via `requests`. Just run them sequentially with asyncio.to_thread().

**Chosen fix:** Replace module-level `_EXECUTOR` with `asyncio.to_thread()` per-collector. This is simpler, no global state, no shutdown requirement.
